

--------------------------------------- DEPLOYING APPLICATION TO HEROKU - DJANGO ---------------------------------------------------

$heroku login --> To login to heroku services
$heroku create --> Creates the app environment, Creates a github repo with remote as 'heroku', Also provides the URL to access the website.


$heroku git:remote -a heroku_app_name --> After creating Heroku app, you can easily add a remote to your local repository 


$pip install gunicorn --> you can't use development server to deploy with heroku so we use gunicorn 
$gunicorn django_project.wsgi --> To run the gunicorn server

Next step is to create a "Procfile" in the django project so that it defines the commands to run once deployed. Create in project root dir.
|web: gunicorn django_project.wsgi
|
|
|:wq


$heroku local --> to deploy on local machine 





STATIC_ROOT = os.path.join(BASE_DIR, 'staticfiles') --> Add a STATIC_ROOT in settings.py file below STATIC_URL. import os required
$heroku config:set DISABLE_COLLECTSTATIC=1 --> To not allow Heroku to run collectstatic on your behalf.
$python manage.py collectstatic --> To collect alstatic files in single folder
$pip install whitenoise --> allow django to serve static images from Appserver. make sure to add to requirements.txt
Goto settings.py then MIDDLEWARE add - 'whitenoise.middleware.WhiteNoiseMiddleware'  and  set DEBUG to 'False'



Now copy heroku app url (eg. "aqueous-caverns-61669.herokuapp.com") and put it in ALLOWED_HOSTS = [] list along with 127.0.0.1 and 0.0.0.0 for local testing



$pip install psycopg2 --> To install library for database. 
Make ENGINE changes in settings.py to 'ENGINE': 'django.db.backends.postgresql',
Then add heroku-postgrese add-on to your app from the app dashboard and then goto the "config vars" from the settings in heroku and add 'NAME', 'HOST', 'PORT', 'USER', 'PASSWORD'
$python manage.py migrate --> to migrate the new DB configurations




$pip freeze > requirements.txt --> Create a requirements.txt file in root of project
$touch runtime.txt --> create runtime file in root of project with python version specified eg. python-3.10.5

$heroku local --> To locally start all of the process types that are defined in your Procfile
$git push origin master --> Make sure to update github branch before updating heroku branch
$git push heroku master --> Adding our project to the heroku remote. Make sure this command is run in root of project folder


NOTE:- Create static and template folder side by side in the app folder

$heroku buildpacks:remove heroku/Lang_Name --> Removing buildpack to an app

$heroku buildpacks:search Lang_Name --> To search for third party buildpacks for a language
